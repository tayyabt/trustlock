# Module Guidance: Approvals

## Responsibilities
- Read/write `.trustlock/approvals.json`
- Create approval entries with input validation
- Validate approval applicability (package, version, rules, expiry)
- Clean expired approvals
- Generate approval commands for output

## Stable Rules
- No wildcard approvals (D9) — `overrides` must list specific rules
- Approver identity from `git config user.name` or `--as` flag (D7)
- Expiry enforced: `default_expiry_days` from config, capped at `max_expiry_days`
- Approval file is a JSON array — append by read + push + rewrite
- `check` reads approvals but never mutates them (Q2)
- Expired approvals are skipped during validation, not deleted

## Usage Expectations
- Read once per `check` invocation
- Written once per `approve` invocation
- Cleaned by `clean-approvals` command only
- Approval commands generated by `generator.js` for every blocked dependency

## Integration Guidance
- Policy engine calls `findValidApproval()` to check if a finding is covered
- CLI `approve` command validates inputs and calls `writeApproval()`
- CLI `clean-approvals` command calls `cleanExpired()`
- Output module includes the generated approval command string in blocked results
